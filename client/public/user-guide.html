1. Signing In

The app uses a 5-digit PIN to identify who you are. Every action you take (adding items, logging charge-outs, etc.) is recorded under your name, so it's important to use your own PIN.

  1. Open the app in your web browser. You will land on the PIN entry screen.
  2. Enter your 5-digit PIN using the on-screen keypad or your keyboard's number keys.
  3. The app signs you in automatically once all 5 digits are entered — no extra button to press.
  4. You are taken to the Dashboard and stay signed in as long as you are actively using the app.
⚠️
Automatic sign-out: The app signs you out after 5 minutes of inactivity to protect against unauthorized access. Simply re-enter your PIN when prompted.

Lockout Protection

After 5 consecutive wrong PIN attempts, the system locks itself. A red "System Locked" screen will appear and no one will be able to sign in until an administrator resets the lockout from the Users page. The screen will warn you after 3 wrong attempts so you can slow down and be careful.

💡
If you forget your PIN, contact your IT administrator. They can delete and recreate your user account with a new PIN.

2. Dashboard

The Dashboard is the first screen you see after signing in. It gives you a quick snapshot of the current state of the inventory.

What You'll See

SectionWhat it shows
Stats cardsTotal items in catalog, low-stock count, this month's charge-out count and total cost, and active loaner count. The loaner card turns red if any loaners are overdue, amber if any are due within 3 days.
Loaner AlertsA warning panel listing any loaner computers that are overdue or due back within 3 days. Click "View all" to go to the Loaners page.
Low Stock AlertsItems whose current stock is at or below their reorder threshold. Highlighted in red as a reminder to reorder.
Low Toner AlertsToner cartridges below their reorder threshold, grouped by printer model and slot color.
Recent Charge-OutsThe last 10 items deployed this month, so you can quickly see recent activity.

Quick Charge-Out

The New Charge-Out button on the Dashboard lets you log an item deployment without leaving the page. A form slides open — fill in the item, department, quantity, and your ticket number, then click Save. The dashboard stats refresh automatically.

3. Inventory

The Inventory page has two tabs: Items (consumables like cables, mice, keyboards, etc.) and Printer Toner. This section covers the Items tab; see Section 4 for Printer Toner.

Reading the Item List

Each row in the table shows one item type:

ColumnMeaning
ItemThe name of the item. A ⚠ icon appears when stock is low.
DescriptionOptional extra detail about the item.
Stock Current quantity on hand.
12 = healthy   2 = at or below reorder threshold
Unit PriceThe price from the most recent purchase order for this item.
Reorder AtWhen stock hits this number or below, the row turns red as a warning.
ℹ️
Stock is always calculated live. It equals total units received (from Purchase Orders) minus total units deployed (from Charge-Outs). There is no manual stock adjustment — everything flows through POs and charge-outs.

Adding a New Item

  1. Click Add Item (top-right of the page).
  2. Enter the item Name (required).
  3. Optionally fill in Description, Unit Price, and Reorder Threshold.
  4. Click Add Item to save.

Editing an Item

Click the pencil icon on any row to open the edit form. Change the fields you need, then click Save Changes.

Deleting an Item

Click the trash icon on any row. You will be asked to confirm. Deleting an item also removes all its associated purchase orders and charge-out records, so do this with care.

Restocking Items

Click the Restock Items button to go directly to the Bulk Purchase Order form, where you can log a new stock delivery for one or many items at once. See Section 6 — Purchase Orders for full details.

4. Printer Toner

The Printer Toner tab (inside Inventory) tracks toner cartridges grouped by printer model. Each printer can have multiple cartridge slots (Black, Cyan, Magenta, Yellow, etc.).

How It's Organized

  • Each printer model appears as a collapsible card.
  • Click the card header to expand or collapse the cartridge list for that printer.
  • A ⚠ 2 low badge on a collapsed printer means some cartridges are running low.

Searching

Use the search bar at the top of the toner tab to filter by any of: printer model name, part number, brand, slot label (e.g. "Cyan"), or notes. Matching printers expand automatically. Clear the search with the ✕ button.

Adding a Printer Model

  1. Click Add Printer (top-right).
  2. Enter the model name (e.g. HP LaserJet M404dn).
  3. Check Color printer if it uses color toner; leave unchecked for black-and-white.
  4. Add optional notes, then click Add Printer.

Adding a Toner Cartridge Slot

  1. Expand a printer card, then click Add Consumable.
  2. Select the Slot (Black, Cyan, Imaging Kit, etc.).
  3. Enter the Part Number and Brand (recommended for ordering).
  4. Set a Reorder Threshold — the stock level at which you want a low-stock warning.
  5. Click Add Cartridge.
ℹ️
Shared part numbers: If two different printers use the same toner (same part number), the system automatically links their stock. Restocking one updates the shared total shown on both. A Shared badge appears on the part number.

Adjusting Stock — Quick Buttons

ButtonWhat it does
+ (plus) next to the stock numberInstantly adds 1 unit to stock (recorded as today's date). Use this for quick single-unit restocks.
− (minus) next to the stock numberOpens the Charge Out Toner form — select department, quantity, and ticket number to log a deployment.
Restock button (right side)Opens the full restock form where you can enter a specific quantity and date received.

5. Departments

Departments represent the areas of the organization that receive IT supplies. Every charge-out must be linked to a department, so this list needs to be kept up to date.

Each Department Record Contains

  • Department Name — e.g. "Accounting", "Nursing"
  • GL Number — the general ledger account number used for billing/reporting

Adding a Department

  1. Click Add Department.
  2. Enter the name and GL number.
  3. Click Add Department to save.

Bulk Import

If you have a list of departments to add at once, use the Bulk Import button. Paste one department per line in the format:

Paste format (one per line)
Department Name, GL Number
Accounting, 10-4001
Human Resources, 10-4015
Nursing 2nd Floor, 20-5102

The system validates every line before saving and reports success or failure for each one individually.

6. Purchase Orders

Purchase Orders (POs) record when new stock arrives. Logging a PO is what increases inventory counts. You can log a single item receipt or an entire order with multiple line items at once.

Single Purchase — "Log Purchase"

  1. Click Log Purchase (top-right).
  2. Select the Item from the dropdown, or switch to Custom to type a one-off item name.
  3. Enter the Date Received, Quantity, and Unit Cost.
  4. Optionally add a PO Number and Notes.
  5. Click Log Purchase. Stock updates immediately.

Bulk Purchase Order — Multiple Items at Once

  1. Click Bulk PO (or click Restock Items from the Inventory page).
  2. Fill in the shared PO Number, Date Received, and optional Notes that apply to the whole order.
  3. Each row in the table is one line item. Select an item, enter quantity and unit cost.
  4. Click + Add Item to add more rows as needed.
  5. Click Submit PO. Each line is saved individually — if one fails, the others still save, and you'll see a summary.
💡
Unit costs are pre-filled from each item's most recent purchase price when you select it — edit them if this order has a different price.

Viewing PO History

Orders are grouped by PO number on the list page. Click a group header to expand it and see the individual line items. The total cost per PO group is shown in the collapsed header.

Deleting a Purchase Order Line

Expand the PO group, then click the trash icon on any line item. This removes that receipt and reduces the item's stock accordingly. Only delete entries that were logged by mistake.

7. Charge-Outs

A Charge-Out records when supplies are deployed or issued to a department. Logging a charge-out reduces the inventory count and creates a billing record.

Logging a Charge-Out

  1. Go to Charge-Outs in the sidebar, then click New Charge-Out.
  2. Select the Item being deployed.
  3. Select the Department receiving it.
  4. Enter the Quantity. The system will warn you if stock is insufficient.
  5. The Unit Cost pre-fills from the item's current price — edit if needed.
  6. Enter a Ticket Number (your helpdesk ticket reference) and the Date.
  7. Charged By is filled in automatically from your login name.
  8. Click Save.
💡
You can also log a charge-out directly from the Dashboard using the Quick Charge-Out button — handy when you're quickly processing a ticket without navigating away.

Filtering the Charge-Out List

Use the month and year dropdowns at the top of the Charge-Outs page to filter the list to a specific period. This makes it easy to review what was deployed in a given month.

Deleting a Charge-Out

Click the trash icon on a row to remove a charge-out. This will return the quantity to inventory. Only delete entries that were logged in error.

8. Loaners

The Loaners page tracks computers that have been temporarily issued to staff. It manages the full lifecycle — from issuing the computer to logging its return — and warns you when due dates are approaching or missed.

The Three Tabs

TabWhat it shows
Active LoanersAll computers currently out on loan, sorted by due date. Shows the borrower's name, department, ticket number, loaned date, due date, and status badge.
Loaner HistoryReturned loaners with the returned date and who logged the return. Read-only; individual records can be deleted if needed.
Manage ComputersThe list of loaner computers you track. Add, rename, or remove computers from here.

Status Badges

Each active loaner shows a status badge based on today's date vs. the due date:

  • Active — due date is more than 3 days away, all is fine.
  • Due Soon — due date is today or within the next 3 days.
  • Overdue — due date has passed and the computer has not been returned.

Issuing a Loaner

  1. Go to Loaners in the sidebar, then click New Loaner (Active Loaners tab).
  2. Select the Computer from the dropdown. Only available (not currently loaned) computers appear.
  3. Select the Department the borrower belongs to.
  4. Enter the Person Name of whoever is borrowing it.
  5. Enter the Ticket # from your helpdesk system (required).
  6. Set the Loaned Date and Due Date.
  7. Add optional Notes, then click Create Loaner.
ℹ️
The Ticket # field is required and renders as a clickable link directly to the helpdesk ticket, making it easy to cross-reference the original request.

Editing an Active Loaner

Click the pencil icon on any active loaner row to open the edit form. You can update any field — person name, department, computer, dates, ticket number, or notes. The edit is audit-logged.

Logging a Return

  1. On the Active Loaners tab, click Log Return on the appropriate row.
  2. A confirmation panel shows the computer name, borrower, and due date.
  3. The Returned By field pre-fills with your name — change it if someone else is processing the return.
  4. Click Confirm Return. The loaner moves to the History tab with today's date as the returned date.

Managing Computers

Before you can issue a loaner, the computer must be in the system. Use the Manage Computers tab to:

  • Add a computer — click Add Computer, enter a name (e.g. GAHL002743) and optional notes such as the model or serial number.
  • Edit — click the pencil icon to rename or update notes.
  • Delete — click the trash icon. Computers currently on active loan cannot be deleted.
⚠️
A computer shown as Currently Loaned cannot be deleted or re-issued until it is returned. Log the return first.

Dashboard Warnings

The Dashboard shows an Active Loaners stat card. Its color reflects the worst status across all active loaners (red for overdue, amber for due soon, blue if all are on schedule). If any loaners are overdue or due soon, a Loaner Alerts panel appears below the stat cards listing each one with the borrower's name, department, and due date.

9. Reports

The Reports page generates a monthly charge-out summary — a full list of everything deployed in a selected month, broken down by department.

Generating a Report

  1. Select the Month and Year using the dropdowns.
  2. The report loads automatically and shows each charge-out line: date, item, department, GL number, quantity, unit cost, and total.
  3. A Department Summary section at the bottom totals up the cost per department for the month.

Exporting to CSV

Click Export CSV to download the monthly report as a spreadsheet-compatible file. This is useful for sharing with finance or pasting into billing systems. The file is named automatically: IT-Inventory-MonthName-Year.csv

10. Users & Security

The Users page is where IT administrators manage who can sign in to the system.

Adding a User

  1. Click Add User.
  2. Enter the person's full name.
  3. Enter a 5-digit PIN for them. Choose something they can remember but others won't guess easily.
  4. Click Add User. The PIN is stored securely (encrypted) — it cannot be retrieved or viewed after saving.
⚠️
PINs cannot be recovered. If a user forgets their PIN, delete their account and create a new one with a new PIN.

Removing a User

Click the trash icon next to a user's name. They will immediately lose access to the system.

Lockout Status

The Users page shows a status card for login security:

  • ✓ No failed attempts — everything is fine.
  • ⚠ N failed attempts — someone entered a wrong PIN recently. The counter resets after a successful login.
  • 🔒 System Locked — 5 consecutive wrong PINs were entered. No one can sign in. Click Reset Lockout to unlock the system.

11. Audit Log

The Audit Log is a record of every change made to inventory items, purchase orders, charge-outs, toner, and loaners — who made it, when, and exactly what changed. It is read-only; entries cannot be edited or deleted.

What Gets Logged

ActionWhat it records
Created A new record was added. Shows all field values at the time of creation.
Updated An existing record was edited (including loaner returns). Shows only the fields that changed, with the old value → new value.
Deleted A record was removed. Shows what it contained before deletion.

Logged Resources

The Resource filter lets you narrow the log to a specific area of the system:

  • Inventory Items — item additions, edits, and deletions
  • Purchase Orders — stock receipts logged or removed
  • Charge-Outs — consumable deployments to departments
  • Printers / Toner Cartridges / Toner Restocks / Toner Charge-Outs — toner inventory changes
  • Loaners — loaner creation, edits, returns, and deletions

Filtering the Log

Use the filter bar at the top to narrow down entries:

  • Resource — filter by a specific resource type (see above)
  • Action — show only Created, Updated, or Deleted entries
  • User — show only changes made by a specific person
  • Date range — pick a preset (Today, Last 7 days, This month, etc.) or enter custom From / To dates

Click Clear filters to reset all filters at once. You can also click a user's name directly in a log entry to instantly filter by that user.

Exporting the Log

Click Export CSV to download the current filtered view as a CSV file. The file includes Timestamp, Resource, Action, Changed By, and a plain-text summary of the change.

12. Tips & FAQ

Frequently Asked Questions

Q: The stock number looks wrong. How do I fix it?

Stock is calculated from purchase orders minus charge-outs. Check the Purchase Orders and Charge-Outs pages to find any entries that shouldn't be there and delete them. Do not delete records just to adjust stock — this removes the historical record. If stock is genuinely off due to a counting discrepancy, log a corrective purchase order (positive adjustment) or a charge-out (negative adjustment) with a note explaining why.

Q: Can I edit a charge-out after saving it?

Charge-outs cannot be edited directly. If an entry is wrong, delete it and re-enter it with the correct information. The deletion and recreation will both appear in the Audit Log.

Q: What is a "custom" item in Purchase Orders?

When logging a purchase, you can switch to Custom mode and type a free-form item name. This is for one-off purchases that don't belong in the tracked inventory (e.g. a single specialty cable). Check Add to inventory if you do want it to appear in the item list and have its stock tracked going forward.

Q: I need to reorder something. Where do I find the part number?

For toner, go to Inventory → Printer Toner, expand the relevant printer, and the Part # column shows the part number recorded for each slot. For consumable items, part numbers can be stored in the item's Description field.

Q: What does "GL Number" mean?

GL stands for General Ledger. It's an accounting code that identifies which budget or cost center gets charged when supplies are deployed to a department. Your finance team manages these codes — make sure the GL number is correct when adding a department.

Q: How do I change my PIN?

PINs cannot be changed after creation. Ask an administrator to delete your account and create a new one with your preferred PIN.

Q: A loaner computer is overdue — what do I do?

The Dashboard will show the overdue loaner in the Loaner Alerts panel and the Active Loaners stat card will turn red. Go to Loaners → Active Loaners, find the row, and either edit the due date if the loan was extended, or click Log Return once the computer is back. Until it is returned it will continue to show as Overdue.

Q: Can I re-issue a loaner to a different person right after it's returned?

Yes. Once a return is logged, the computer immediately becomes available again and will appear in the dropdown when creating a new loaner.

Q: The loaner computer I want to delete is greyed out. Why?

A computer that is currently on active loan cannot be deleted. Log the return first, then delete it from Manage Computers.

13. Updating the App

When a new version is released, follow the steps below for your installation type. Your database is never touched by an update — all existing data is preserved.

Docker installation (Linux server)

Run these commands on the server:

cd /path/to/InventoryApp
git pull
docker compose up -d --build

git pull downloads the latest code from GitHub. docker compose up -d --build rebuilds the container image and restarts the app. The data/ folder (your database) is volume-mounted and is never affected.

Windows / Node + PM2 installation (no Docker)

Open PowerShell in the app folder and run:

cd C:\InventoryApp
git pull
npm run build
pm2 restart inventory

git pull fetches the latest code. npm run build rebuilds the React frontend into client\dist\ — this step is required whenever the frontend changes. pm2 restart inventory restarts the Express server to pick up any backend changes.

💡
If you're not sure which installation type you have: if the app is running inside Docker, use the Docker steps. If it was installed with Node.js and PM2 directly on Windows, use the Windows steps.
